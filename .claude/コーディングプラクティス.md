# コーディングプラクティス

## スキルスタック

- FW
  - React Router
- フロントエンド
  - 言語
    - Typescript
  - UI
    - React
    - shadcn/ui
    - Tailwind CSS v4
- バックエンド
  - 実行環境
    - Node.js
  - 言語
    - Typescript

## 関数型アプローチ

- 純粋関数を優先
- 不変データ構造を使用
- 副作用を分離
- 型安全性を確保

## ドメイン駆動設計 (DDD)

- 値オブジェクトとエンティティを区別
- 集約で整合性を保証
- リポジトリでデータアクセスを抽象化
- 境界付けられたコンテキストを意識

## テスト駆動開発 (TDD)

- Red-Green-Refactorサイクル
- テストを仕様として扱う
- 小さな単位で反復
- 継続的なリファクタリング

## コードスタイル

- 早期リターンを重視
- 小さく始めて段階的に拡張
- 過度な抽象化を避ける
- コードよりも型を重視
- 複雑さに応じてアプローチを調整
- 関数優先（クラスは必要な場合のみ）
- 不変更新パターンの活用
- エラーとユースケースの列挙型定義
- 1ファイル500行以内を目標にする

### 型設計

- まず型を定義
- ドメインの言語を型で表現

### 純粋関数から実装

- 外部依存のない関数を先に
- テストを先に書く

### 副作用を分離

- IO操作は関数の境界に押し出す
- 副作用を持つ処理をPromiseでラップ

### アダプター実装

- 外部サービスやDBへのアクセスを抽象化
- テスト用モックを用意
